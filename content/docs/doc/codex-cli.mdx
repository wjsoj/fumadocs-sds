## 这篇教程能帮你做什么？

**Codex** 是 OpenAI 推出的命令行编程助手（Coding Agent）。配置好之后，你可以在终端里输入一句话，
让它帮你读代码、写代码、改 Bug、跑命令。本教程会**手把手**带你完成全部步骤，即使你从没用过命令行也能跟着走通。

<Callout type="info" title="开始之前，先看这 3 句话">
1. 全程大约需要 **10～20 分钟**，请保持网络畅通。
2. 下面所有 <strong>灰色代码块</strong> 里的命令，都是「复制 → 粘贴到终端 → 按回车」。
3. 页面顶部的 <strong>Windows / macOS / Linux</strong> 选项卡会整页联动，**先选好你自己的电脑系统**，然后从上往下照做即可。
</Callout>

<Callout type="warn" title="名词速查（看不懂时回来对一下）">
- **终端 / 命令行**：一个让你用「打字」代替「点鼠标」来控制电脑的黑色窗口。Windows 上叫 *PowerShell* 或 *CMD*，macOS / Linux 上叫 *Terminal（终端）*。
- **Node.js**：一个运行环境，Codex 依赖它才能安装和运行。
- **npm**：随 Node.js 一起安装的「软件管家」，用来下载 Codex。
- **API 令牌（sk-xxx）**：相当于一把钥匙，让 Codex 能调用模型。**它等同于密码，不要发给任何人。**
</Callout>

## 第 1 步：安装 Node.js

<Tabs groupId="os" persist items={['Windows', 'macOS', 'Linux']}>

<Tab value="Windows">

#### 1.1 先安装 Git Bash（仅 Windows 需要）

打开 [Git 官方下载页](https://git-scm.com/downloads/win)，点击页面上的 **「Click here to download」** 下载安装包，
双击运行后 <strong>一路点「Next（下一步）」</strong> 直到完成即可，不需要修改任何选项。

![Git for Windows 官方下载页面](https://img-api.pku3d.com/ota/admin/hMhUeymH.png)

#### 1.2 安装 Node.js

打开 [Node.js 官网](https://nodejs.org/en/download)，下载并安装标有 **LTS** 字样的版本（LTS = 长期支持版，最稳定）。
下载得到的 `.msi` 安装包同样 <strong>一路点「Next」</strong> 完成安装即可。

![Node.js 官方下载页面](https://img-api.pku3d.com/ota/admin/UHcTwP7M.png)

<Callout type="info">
安装界面里如果看到 “Add to PATH（添加到环境变量）” 的勾选项，请**保持勾选**，这样命令行才能找到 node 命令。
</Callout>

</Tab>

<Tab value="macOS">

有两种方式，**任选其一**即可。

**方式一（最简单）：官网下载安装包**

打开 [Node.js 官网](https://nodejs.org/en/download)，下载标有 **LTS** 字样的 `.pkg` 安装包，双击后一路点「继续」完成安装。

![Node.js 官方下载页面](https://img-api.pku3d.com/ota/admin/UHcTwP7M.png)

**方式二（推荐，命令行一步到位）：用 Homebrew**

打开「终端」（在「启动台」搜索 *Terminal* 即可），先安装 Homebrew（如果你电脑里还没有）：

```bash
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
```

再安装 Node.js：

```bash
brew install node
```

</Tab>

<Tab value="Linux">

打开终端，根据你的发行版选择对应命令复制执行：

**Ubuntu / Debian**

```bash
sudo apt update
curl -fsSL https://deb.nodesource.com/setup_lts.x | sudo -E bash -
sudo apt-get install -y nodejs
```

**CentOS / RHEL / Fedora**

```bash
sudo dnf install nodejs npm   # 较新的系统
# 如果上面的命令找不到，改用：
sudo yum install nodejs npm
```

**Arch Linux**

```bash
sudo pacman -S nodejs npm
```

</Tab>

</Tabs>

## 第 3 步：验证安装是否成功

在终端里运行：

```bash
codex --version
```

如果屏幕上打印出一串**版本号**（例如 `codex 0.x.x`），说明安装成功 🎉。
如果提示「command not found / 不是内部或外部命令」，请**关掉终端重新打开**再试一次；仍然不行就回到第 1、2 步重新检查。

## 第 4 步：准备 API 令牌与接口地址

Codex 需要两样东西才能工作：

| 名称 | 值 | 作用 |
| --- | --- | --- |
| **接口地址（Base URL）** | `https://openai.phybench.cn/v1` | 告诉 Codex 连到哪台服务器（已为你配好，下一步配置里直接是它） |
| **API 令牌（Token）** | `sk-xxxxxxxxxxxxxxxx` | 调用模型的「钥匙」，**会单独提供给你** |

接口地址是固定的；你只需要拿到属于你自己的**令牌**，下一步填进配置文件即可。

<Callout type="warn">
`sk-` 开头的令牌等同于密码。**不要截图发群、不要提交到 GitHub、不要告诉别人。**
</Callout>

## 第 5 步：写入配置文件

我们要在「用户目录」下创建一个 `.codex` 文件夹，里面放两个文件：

- `auth.json` —— 存放你的密钥
- `config.toml` —— 告诉 Codex 用哪个模型、连哪个服务器

<Tabs groupId="os" persist items={['Windows', 'macOS', 'Linux']}>

<Tab value="Windows">

**方法 A：用命令行创建（推荐，最省事）**

打开 **PowerShell**，逐行复制运行：

```powershell
# 1. 创建 .codex 文件夹
mkdir "$HOME\.codex" -Force

# 2. 写入 auth.json（请先把 sk-xxx 换成你自己的真实密钥！）
'{"OPENAI_API_KEY": "sk-xxx"}' | Set-Content "$HOME\.codex\auth.json"

# 3. 用记事本打开 config.toml 准备粘贴内容
notepad "$HOME\.codex\config.toml"
```

记事本弹出后（提示是否新建文件，点「是」），把下面这段**原样粘贴进去**，然后保存：

```toml
model_provider = "api111"
model = "gpt-5-codex"
model_reasoning_effort = "high"
disable_response_storage = true
preferred_auth_method = "apikey"

[model_providers.api111]
name = "api111"
base_url = "https://openai.phybench.cn/v1"
wire_api = "responses"
```

**方法 B：手动创建文件**

进入 `C:\Users\你的用户名\.codex` 目录（看不到这个文件夹时，先在文件资源管理器「查看」里勾选**显示隐藏的项目**），
手动新建 `auth.json` 和 `config.toml` 两个文件，分别填入上面的内容。

</Tab>

<Tab value="macOS">

打开「终端」，逐行复制运行：

```bash
# 1. 创建文件夹和两个空文件
mkdir -p ~/.codex
touch ~/.codex/auth.json
touch ~/.codex/config.toml

# 2. 编辑 auth.json
open -e ~/.codex/auth.json
```

在弹出的文本编辑器里粘贴下面内容（**把 sk-xxx 换成你的真实密钥**），保存关闭：

```json
{"OPENAI_API_KEY": "sk-xxx"}
```

再编辑 config.toml：

```bash
open -e ~/.codex/config.toml
```

粘贴下面内容并保存：

```toml
model_provider = "api111"
model = "gpt-5-codex"
model_reasoning_effort = "high"
disable_response_storage = true
preferred_auth_method = "apikey"

[model_providers.api111]
name = "api111"
base_url = "https://openai.phybench.cn/v1"
wire_api = "responses"
```

</Tab>

<Tab value="Linux">

打开终端，逐行复制运行：

```bash
# 1. 创建文件夹和两个空文件
mkdir -p ~/.codex
touch ~/.codex/auth.json
touch ~/.codex/config.toml

# 2. 用 vi 编辑 auth.json
vi ~/.codex/auth.json
```

进入 vi 后：按一下 **`i`** 进入编辑模式，粘贴下面内容（**把 sk-xxx 换成你的真实密钥**），
然后按 **`ESC`**，输入 **`:wq`** 再回车保存退出：

```json
{"OPENAI_API_KEY": "sk-xxx"}
```

再编辑 config.toml：

```bash
vi ~/.codex/config.toml
```

同样按 `i` 进入编辑，粘贴下面内容，按 `ESC` 输入 `:wq` 保存：

```toml
model_provider = "api111"
model = "gpt-5-codex"
model_reasoning_effort = "high"
disable_response_storage = true
preferred_auth_method = "apikey"

[model_providers.api111]
name = "api111"
base_url = "https://openai.phybench.cn/v1"
wire_api = "responses"
```

</Tab>

</Tabs>

<Callout type="info" title="config.toml 里几个值是什么意思？">
- `model_reasoning_effort` 可选 `high` / `medium` / `low`，代表模型「思考」的努力程度（越高越仔细但越慢）。
- `base_url` 是接口地址，已填好为 `https://openai.phybench.cn/v1`，**保持不动即可**。
- `model = "gpt-5-codex"` 是默认使用的模型。
</Callout>

## 常见问题（FAQ）

<Callout type="error" title="运行 codex 报错 / 无法使用？按顺序排查">
1. **检查令牌**：`auth.json` 里的 `sk-xxx` 必须换成提供给你的真实令牌，不要多空格、不要漏引号；`config.toml` 里的 `base_url` 保持 `https://openai.phybench.cn/v1` 不要改。
2. **检查是否重启了终端**：改完配置一定要开新窗口。
3. **检查 Node 版本**：运行 `node -v`，需为 22 及以上；过低请回到第 1 步重装 LTS 版本。
4. **仍不行**：把令牌、接口地址、额度等信息向你的服务提供方再确认一遍。
</Callout>

**Q：提示 `codex: command not found`？**
A：多半是没重启终端，或 Node.js 没装好 / 没加入 PATH。重开终端，再依次运行 `node -v`、`npm -v`、`codex --version` 定位是哪一步出了问题。

**Q：`config.toml` 放哪里？**
A：放在用户目录的 `.codex` 文件夹里。Windows 是 `C:\Users\你的用户名\.codex\`，macOS / Linux 是 `~/.codex/`。

> 更多 Codex 的配置与用法，可参考 [Codex 官方文档](https://developers.openai.com/codex/ide)。
